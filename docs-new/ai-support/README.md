# AI 辅助开发

本指南帮助你使用 AI 快速生成符合 ErisPulse 规范的模块和适配器代码。

## AI 物料选择

根据你的生成需求选择合适的文档：

| 文档类型 | 适用场景 | 推荐度 |
|---------|----------|--------|
| **模块开发物料** | 模块开发 | ⭐⭐⭐⭐⭐ |
| **适配器开发物料** | 适配器开发 | ⭐⭐⭐⭐⭐ |
| **完整物料集合** | 完整开发参考（仅限大模型） | ⭐⭐ |

### 文档获取方式

所有 AI 物料文档位于 `ai-support/prompts/` 目录下：

- `module-dev.md` - 模块开发专用物料
- `adapter-dev.md` - 适配器开发专用物料

> 💡 **提示**：
> - 对于大多数 AI 模型，推荐使用专用的 Module/Adapter 文档
> - 建议在提示词中明确告知 AI 使用哪个文档

## 模块开发指南

### 模块结构

一个标准的模块包结构：

```
MyModule/
├── pyproject.toml
├── README.md
├── LICENSE
└── MyModule/
    ├── __init__.py
    └── Core.py
```

## 常见问题

### Q: 推荐使用什么 AI 工具？

A: 推荐使用以下 AI 工具：
- **VS Code + GitHub Copilot**：可以直接操作文件系统
- **Cursor**：强大的 AI 编码助手
- **ChatGPT / Claude**：可以通过对话生成代码

### Q: AI 生成的代码有问题怎么办？

A: 建议按以下步骤处理：
1. 检查 AI 是否使用了正确的文档版本
2. 向 AI 提供更详细的需求描述
3. 让 AI 分步生成，逐步验证
4. 参考示例模板，提供更具体的示例

### Q: 如何让 AI 生成更高质量的代码？

A: 建议：
1. 提供详细的需求描述，包括所有细节
2. 提供具体的示例，包括输入和输出
3. 明确指定使用的文档版本
4. 要求 AI 添加注释和文档字符串
