# AI 辅助开发

本指南帮助你使用 AI 快速生成符合 ErisPulse 规范的模块和适配器代码。

## 📦 AI 物料文档

ErisPulse 提供了完整的 AI 开发物料文档，可以直接投喂给 AI 生成符合规范的代码。

### 文档类型

| 文档名称 | 适用场景 | 内容规模 | 推荐度 |
|---------|----------|---------|--------|
| **ErisPulse-ModuleDev.md** | 模块开发 | 中等 | ⭐⭐⭐⭐⭐ |
| **ErisPulse-AdapterDev.md** | 适配器开发 | 中等 | ⭐⭐⭐⭐⭐ |
| **ErisPulse-Full.md** | 完整开发参考 | 大型 | ⭐⭐⭐ |

### 获取物料文档

#### 方法一：使用生成脚本（推荐）

```bash
# 从项目根目录执行
python scripts/tools/generate-ai-prompts.py
```

生成的文档位于 `docs-new/ai-support/prompts/` 目录：
- `ErisPulse-ModuleDev.md` - 模块开发专用物料
- `ErisPulse-AdapterDev.md` - 适配器开发专用物料
- `ErisPulse-Full.md` - 完整开发物料

#### 方法二：从 GitHub Releases 下载

每次发布新版本时，会在 GitHub Releases 中提供最新的 AI 物料文档。

```

├── README.md
```

## 常见问题

### Q: 推荐使用什么 AI 工具？

A: 推荐使用以下 AI 工具：
- **VS Code + GitHub Copilot**：可以直接操作文件系统
- **Cursor**：强大的 AI 编码助手
- **ChatGPT / Claude**：可以通过对话生成代码

### Q: AI 生成的代码有问题怎么办？

A: 建议按以下步骤处理：
1. 检查 AI 是否使用了正确的文档版本
2. 向 AI 提供更详细的需求描述
3. 让 AI 分步生成，逐步验证
4. 参考示例模板，提供更具体的示例

### Q: 如何让 AI 生成更高质量的代码？

A: 建议：
1. 提供详细的需求描述，包括所有细节
2. 提供具体的示例，包括输入和输出
3. 明确指定使用的文档版本
4. 要求 AI 添加注释和文档字符串
